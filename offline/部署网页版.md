# 网页版部署说明

圆形二维码生成器 **本身就是网页应用**（`offline/`），与 Mac 桌面版共用同一套界面，只是桌面版多了一层 pywebview 外壳。

## 一、本机立即使用（局域网 / 本机）

```bash
cd offline
npm install   # 首次
npm run build
npm run preview
```

浏览器打开：

- 本机：**http://127.0.0.1:5173**
- 同一 WiFi 下的手机：终端会显示 `Network: http://192.168.x.x:5173`，用手机浏览器访问该地址

> 不要用 `file://` 直接打开 `dist/index.html`，部分功能会受限。

## 二、部署到公网（输入网址即可访问）

### 方式 A：Vercel（推荐，免费 HTTPS）

1. 注册 [vercel.com](https://vercel.com)，安装 CLI：`npm i -g vercel`
2. 在项目里执行：

```bash
cd offline
npm run build
vercel --prod
```

3. 或在 Vercel 网站 **Import Git Repository**，设置：
   - **Root Directory：** `offline`
   - **Build Command：** `npm run build`
   - **Output Directory：** `dist`

4. 部署完成后会得到类似 `https://xxx.vercel.app` 的网址，可直接分享。

### 方式 B：GitHub Pages（免费）

1. 将代码 push 到 GitHub 仓库（如 `wudicow666/linkme.ltd`）
2. 仓库 **Settings → Pages → Source** 选 **GitHub Actions**
3. push 到 `main` 分支后，工作流 `.github/workflows/deploy-web.yml` 会自动构建并发布
4. 访问地址：`https://<用户名>.github.io/<仓库名>/`

例如：`https://wudicow666.github.io/linkme.ltd/`

### 方式 C：任意静态托管

构建产物在 `offline/dist`，上传到：

- 阿里云 OSS / 腾讯云 COS（开启静态网站）
- Netlify、Cloudflare Pages 等

## 三、URL 参数（可选）

| 参数 | 示例 | 说明 |
|------|------|------|
| `url` | `?url=https://work.weixin.qq.com/ca/xxx` | 预填获客链接 |
| `auto` | `?auto=1` | 隐藏顶栏，适合嵌入 |

示例：`https://你的域名/?url=https://work.weixin.qq.com/ca/cawcdeeae226151871`

## 四、与桌面 App 的区别

| 功能 | 网页版 | Mac 桌面 App |
|------|--------|--------------|
| 圆码生成 / 560 样式 / 模板 | ✅ | ✅ |
| 导出 PNG | 浏览器下载 / 手机分享 | 系统「另存为」 |
| 需安装 | ❌ 只需浏览器 | ✅ .app |
| 离线使用 | 打开页面后本地计算，首次需加载网页 | 完全离线 |

## 五、手机浏览器

- 布局在窄屏会自动变为上下排列
- 导出时 iPhone 可能弹出 **分享菜单**，选「存储到照片」即可
- 扫码测试请用 **扫一扫 → 相册选图**（见工作记录 §13.7）
