## 1. 项目目标

为企业微信**获客链接**生成可扫描的**圆形方格二维码**，并可套入**抖音圆码海报模板**，在本地离线完成预览与 PNG 导出。支持：

- 双击桌面 App 使用（Mac `.app` / Windows 解压即用）
- 无需登录、无需联网（生成与导出均在本地）
- 中文界面
- 模板内手动拖拽、缩放圆码，**并可输入 X/Y 数值精调位置**
- 可选在海报右下角添加自定义文字（字号、颜色可调）

技术路线参考 [linkme.ltd](https://github.com/AWKohler/linkme.ltd) 的圆码样式，使用 **`qrcode-generator` + 自研 `CircularQRCode`**（不使用 npm 的 `qrcode` 包）。

## 3. 系统架构

```
┌─────────────────────────────────────────────────────────┐
│  桌面壳 desktop/launcher.py (pywebview + 本地 HTTP)        │
│  ├─ DesktopApi：系统「另存为」、分块接收大图               │
│  └─ 托管 offline/dist 静态资源                           │
└──────────────────────────┬──────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────┐
│  前端 offline/ (Vite) — 复用上层 components/、lib/        │
│  ├─ QRCodeGenerator.tsx      主界面（标签页、状态）       │
│  ├─ CircularQRCode.tsx         圆形方格 SVG 绘制          │
│  ├─ TemplateManualComposer.tsx 模板预览（拖拽/数值调位/缩放）      │
│  └─ GradientPicker/            渐变色标等                 │
└──────────────────────────┬──────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────┐
│  lib/                                                    │
│  ├─ shellTemplates.ts        模板元数据（抖音海报）       │
│  ├─ exportManualShell.ts     Canvas 合成导出             │
│  ├─ shellOverlayText.ts      海报角标文字绘制           │
│  ├─ qrRasterize.ts           SVG → PNG                   │
│  └─ downloadFile.ts          浏览器下载 / 桌面保存       │
└─────────────────────────────────────────────────────────┘
```

**数据流（模板导出）**

1. 用户在「基础」输入链接 → `qrcode-generator` 生成矩阵 → `CircularQRCode` 渲染 SVG  
2. 「模板」选抖音海报 → `TemplateManualComposer` 叠加实时 SVG 预览  
3. 「导出」→ `exportManualShellComposite` 合成模板图 + 圆码 + 可选角标文字 → `downloadFile` → 系统保存对话框

## 5. 功能说明（当前版本）

### 5.1 基础

- 输入企业微信获客链接或任意文本  
- 二维码尺寸、**裁成圆形**及裁切半径  
- **圆码紧凑度**（82%–100%，默认 **90%**）：等比缩小圆码、增大外圈点阵占比；低于 85% 界面提示可能影响扫码  
- 链接变更后预览**实时更新**（含模板模式）

### 5.2 样式 / 图案 / 边框 / 图片

- 背景、码点颜色、圆角、缩放、外圈边框、中心图、环形装饰条等（继承 linkme 能力）  
- 界面文案已中文化  
- **定位角样式**（「图案」标签页）：
  - **560 定位角（推荐，默认）**：每个 7×7 定位角 **主轴 17 点**（中心十字 5 点 + 四边各 3 点）；开启「扫一扫优化」时另补 **4 角点**（不补满框）
  - **十字（弱化外框）**：全量绘制 finder 暗模块，非十字部分缩小+半透明；备用
  - **圆点方框** / **经典三层方块**：备用样式
- **扫一扫优化**（`scanOptimized`，默认 **开**，仅 `sparse560` 下显示）：
  - 对齐块：**中等强度**（scale 0.93，opacity 0.96），兼顾微信/抖音扫一扫
  - 定位角：17 主轴点（scale 1）+ 四角补点（scale 0.96）；**不**补满 7×7（避免方框感）
  - 关闭后更接近纯 560 留白，抖音扫一扫可能失败
- **560 视觉栈**（`cross` / `sparse560` 共用）：数据区与背景区同色实心圆点；背景点**严禁**进入 QR 矩阵 bbox；QR 外缘过渡环加密；外圈 3 小靶心 + 2 点大圆环装饰

### 5.3 模板（抖音圆码海报）

- 左侧：海报底图 + 可**拖动**圆码 + 实时 SVG  
- 控件（预览下方控制区）：
  - 圆码直径 **80–500px**（滑块 + 数字）
  - **水平位置 X**、**垂直位置 Y**（模板像素，**圆心坐标**；范围 X：0–743，Y：0–1024；拖动与输入双向同步）
  - **重置位置与大小**（恢复模板默认 **X=368，Y=344，直径 324**）
- **在海报上添加文字**：开关 + 内容 + 字号(12–72) + 颜色；显示在海报**右下角**，预览与导出一致  

### 5.4 导出

- 无模板：导出 PNG / WEBP / SVG（可选分辨率、导出内容）  
- 有模板：导出**当前合成海报** PNG（建议格式 PNG）  
- **Mac/Windows 桌面 App**：系统「另存为」；大图走分块 API  
- 提示区分：**已保存** / **已取消保存** / **保存失败：原因**  
- **扫一扫使用建议（§13.7）：** PNG、分辨率 ≥1280；保存相册后 **微信/抖音「扫一扫 → 相册选图」**；聊天发图长按 **不保证**

## 7. 桌面导出技术说明（排错用）

### 7.1 为何不能用 `<a download>`

pywebview 内嵌 WebView 对 data URL 下载支持差，故由 Python `DesktopApi` 弹出原生保存对话框写文件。

### 7.2 `save_data_url` 传参

- 前端使用 **JSON 单参数**：`JSON.stringify({ data, name })`，避免 pywebview 将多参数打成 **tuple** 导致 `re.match` 报错。  
- 大图（base64 较长）使用 **`save_image_begin` / `save_image_append` / `save_image_finish`** 分块传输。  

### 7.3 返回值

Python 返回 `{ ok, path }` / `{ cancelled: true }` / `{ error: "..." }`，前端 `downloadResultMessage()` 映射为中文提示。

## 13. 560 风格圆码攻坚（2026-06-05）

> **任务状态：已全部完成并验收**  
> 视觉：560 稀疏十字定位角 ✔ · 微信扫一扫 ✔ · 抖音扫一扫 ✔ · 聊天长按 ✘（不要求）

### 13.1 目标与参照

- **参照样张：** 抖音圆码海报风格（用户称「560」；完整图含外圈靶心、虚线弧、中心微信 Logo、底栏文案；矩阵三角区为 **十字 + 四边各 3 点** 稀疏定位角）
- **编码内容：** 企业微信获客链接（`https://work.weixin.qq.com/ca/...`）
- **主使用场景：** 抖音；**最终验收口径（§13.7）** 以双端 **扫一扫 → 相册选图** 为准

### 13.2 问题分析（为何不能「直接 clone 560」）

| 现象 | 原因 |
|------|------|
| 560 定位角极「空」仍能扫 | 标准 QR 数据 + stylized 外观；平台扫一扫对圆码较宽容 |
| 早期「只画十字、去掉方框数据」 | 矩阵定位结构缺失，**扫不出** |
| 背景实心黑点画进 QR 留白 | 白格被污染，**扫不出** |
| 补满 7×7 定位角暗点 | 抖音/微信能扫，但视觉变 **方框点阵**，用户否决 |
| 对齐块全实心 + 稀疏定位角 | 抖音扫一扫 ✔，微信扫一扫 ✘（组合互斥） |

结论：在 **保留标准 `qrcode-generator` 编码** 前提下，采用 **560 稀疏 17 点 + 扫一扫优化（四角 + 平衡对齐块）**，在观感与双端扫一扫之间折中。

### 13.3 实施方案（分阶段，均已完成）

**阶段一 — 圆码紧凑度 `qrCompactness`**

- 范围 82%–100%，默认 **90%**；`effectiveQrCodeSize = qrCodeSize × compactness`
- 缩小圆码、增大外圈点阵；UI 在「基础」标签页

**阶段二 — 560 视觉栈**

- 数据点：scale **0.96**，opacity **1**
- 背景点：与数据点同色，**仅在 QR 方阵外**（`distanceToQrBoundingBox`）
- 过渡环：距 QR 外缘 &lt;4.5 module，背景点出现率 **72%**
- 外圈装饰：10/4/8 点靶心 + 2 点大圆环（`cross` / `sparse560` 共用）

**阶段三 — 560 稀疏定位角 `sparse560`**

每个 7×7 定位角 **主轴 17 格**（`SPARSE560_FINDER_LOCAL`）：

```
十字: (2,3)(3,3)(4,3)(3,2)(3,4)
上:   (0,2)(0,3)(0,4)
下:   (6,2)(6,3)(6,4)
左:   (2,0)(3,0)(4,0)
右:   (2,6)(3,6)(4,6)
```

- 主轴 17 点：`getSparse560AccentVisual()` → scale **1**，opacity **1**
- 关闭「扫一扫优化」时：其余 finder 暗模块 **不绘制**

**阶段 3.5 — 扫一扫优化 `scanOptimized`（默认开启，最终版）**

| 模块 | 行为 |
|------|------|
| 定位角四角 | `SPARSE560_SCAN_CORNER_LOCAL`：(0,0)(0,6)(6,0)(6,6)，scale 0.96，**不补满框** |
| 对齐块 | `getScanBalancedAlignmentVisual()`：scale **0.93**，opacity **0.96**（非全实心、非弱化） |
| 禁止项 | **不**补满 7×7 finder（会变方框）；**不**将对齐块补成 scale 0.96 全实心（会导致微信扫一扫失效） |

UI：「图案」→ 开关「扫一扫优化（推荐）」；「导出」→ 提示 PNG ≥1280、相册选图。

## 14. 模板圆码位置数值精调（2026-06-05）

> **任务状态：已完成** — 用户可在套抖音海报时除拖拽外，用 X/Y 数值精确对齐圆环区；默认圆心已校准。

### 14.1 需求背景

- 已有 `TemplateManualComposer` 支持**拖动**圆码、滑块/数字调整**直径**
- 用户希望增加 **水平 / 垂直位置数值输入**，便于 repeatable 对齐模板留白/圆环（无需纯靠手感拖拽）
- 实测后确定抖音海报 (`douyin-shell`) 更合适默认圆心 **(368, 344)**（原为 371/358）

### 14.2 实现说明

| 项 | 说明 |
|----|------|
| **坐标含义** | `qrX` / `qrY` 为圆码**圆心**在模板画布上的像素坐标（与拖动逻辑一致） |
| **X 输入** | 0–`template.width`（743），整数，变更即更新预览 |
| **Y 输入** | 0–`template.height`（1024），整数 |
| **与拖动关系** | 拖动圆码时 X/Y 数字同步；改数字时预览即时移动 |
| **重置** | 「重置位置与大小」读取 `shellTemplates.ts` 中 `defaultQrX/Y/Diameter` |
| **导出** | `exportManualShellComposite` 仍使用当前 `shellTransform`，与预览一致 |

实现位置：`TemplateManualComposer.tsx` 内 `clampCoord()` + 双列数字输入框（与直径输入 UI 风格一致）。

### 14.3 默认值变更

| 字段 | 旧值 | 新值 |
|------|------|------|
| `defaultQrX` | 371 | **368** |
| `defaultQrY` | 358 | **344** |
| `defaultQrDiameter` | 324 | 324（不变） |

同步修改：

- `lib/shellTemplates.ts` — 模板元数据  
- `components/QRCodeGenerator.tsx` — `shellTransform` 初始 state；切换模板时 `useEffect` 仍读模板 default  
- 「模板」标签页说明文案

### 14.4 改动文件

| 文件 | 变更 |
|------|------|
| `components/TemplateManualComposer.tsx` | 新增 X/Y 数字输入；更新底部说明 |

| `components/QRCodeGenerator.tsx` | 初始 transform；模板 Tab 提示文案 |

### 14.5 使用步骤

1. 「模板」→ 选择 **抖音圆码海报**  
2. 左侧预览下方调整 **水平 X / 垂直 Y**（或拖动圆码）

## 15. 网页版（浏览器输入网址访问）（2026-06-05）

> **说明：** 圆形二维码生成器 **本来就是网页应用**（`offline/`）；Mac `.app` 只是在 pywebview 里打开同一套页面。无需重写即可部署为公网网址。

### 15.1 架构关系

```
components/ + lib/  ←── 共用业务代码
        │
        ├── offline/ (Vite) ──→ 浏览器 / 公网 URL
        └── desktop/ (pywebview) ──→ Mac 桌面 App
```

- 浏览器导出：`lib/downloadFile.ts` 走 `<a download>`；手机支持 **Web Share**（存相册）
- 桌面导出：走 `DesktopApi` 系统保存对话框

### 15.2 本地 / 局域网使用

```bash
cd offline && npm install && npm run build && npm run preview
```

| 访问方式 | 地址 |
|----------|------|
| 本机 | http://127.0.0.1:5173 |
| 同 WiFi 手机 | 终端显示的 `Network: http://192.168.x.x:5173` |

根目录也可：`npm run offline:preview`

### 15.3 公网部署（输入网址直接进入）

| 方式 | 说明 | 文档 |
